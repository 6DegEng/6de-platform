# Session 3c — SharePoint Mirror

**Goal:** Auto-generate two kinds of SharePoint-resident snapshots of platform state so anyone with OneDrive access can read project info without opening the platform:

1. **Per-project `_AUTO_project_summary.md`** in each project's SharePoint folder. Human-readable markdown. Refreshed on every project mutation.
2. **Portfolio-wide `_AUTO_portfolio_overview.xlsx`** at the active projects root. Mirrors the legacy `Project_Tracker_2026.xlsx` column shape so it's instantly recognizable. Refreshed on schedule or on demand.

**Branch:** create `feature/sharepoint-mirror` off the post-3b state.

**Pre-conditions:**
- Session 3b is merged (the markdown summary needs notes/contacts/updates/priority/% complete to exist).
- SharePoint Phase C is merged (real Graph SDK wiring is needed to write to SharePoint).
- Platform boots clean.
- `.env` has working MSGRAPH_CLIENT_ID, MSGRAPH_TENANT_ID, SIXDE_TOKEN_KEY.

**Estimated time:** 4–6 hours autonomous.

## Notes for Juan

- **The trigger architecture is the riskiest piece.** Streamlit's rerun model + background threads + SharePoint API throttling can get messy. The 60-second debounce is conservative on purpose — better to wait a minute and batch than to thrash SharePoint with one upload per keystroke. If it feels too slow, lower it after the system proves stable, not before.
- **The legacy xlsx column shape is intentional.** Anyone who's used `Project_Tracker_2026.xlsx` will recognize `_AUTO_portfolio_overview.xlsx` instantly. That's the UX win — it lowers the bar to "this is the new tracker" because it looks like the old one. The "AUTO" prefix and red banner are the only structural changes.
- **Idempotency tests will catch openpyxl's quirks.** openpyxl can produce non-deterministic xlsx output (zip metadata order, internal IDs). The tests will surface that. There are known workarounds (e.g. set zipfile timestamps to a fixed epoch); the spec-writer should document them.
- **You can read the markdown summaries on your phone.** OneDrive's mobile app renders markdown decently. So when you're at a job site and need to remember what's going on with a project, the SharePoint folder has a readable summary without you needing the platform installed on your phone.
- **The financial fields might be empty initially.** If your platform doesn't yet have a financial roll-up (contract/paid/outstanding/COGS/profit), the spec-writer will surface that gap. You can either backfill the financials manually in the legacy importer (Session 3b), or punt the financial columns to a later session and ship the markdown + non-financial xlsx now.
- **After 3c lands, the legacy `Project_Tracker_2026.xlsx` becomes a historical artifact.** Don't delete it (it's history), but rename to something like `_LEGACY_Project_Tracker_2026.xlsx` and put it in an archive folder so the platform's `_AUTO_portfolio_overview.xlsx` becomes the only "current" tracker. That's a manual cleanup task, separate from the session.
